# Changelog

本文按版本记录 Secrel 的用户可见变化，包括功能、行为、接口和会影响用户理解的文档更新。开发期间先写入 `Unreleased`；发布时创建具体版本段落，并把对应条目从 `Unreleased` 移入该版本。

内部讨论、未接受 proposal、临时实验和不影响用户使用方式的重构不需要写入本文。

条目类型：

- `feat`：新增用户可见能力。
- `fix`：修复用户可见问题。
- `changed`：改变已有行为、配置、接口或交互。
- `docs`：影响用户理解或使用方式的文档变化。
